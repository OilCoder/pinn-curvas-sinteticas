## 10:00 — Diagnóstico inicial del estado del preprocesamiento

### Hallazgos

Se revisó `src/preprocessing.py`, `src/data_loader.py` y `tests/test_preprocessing.py` antes de proponer cambios.

**Problemas críticos encontrados:**

1. **NPHI en porcentaje** — Todos los LAS del campo Kraft Prusa almacenan CNLS en % (valores 14–31), no en v/v. El pipeline lo ingería sin corregir → media de NPHI = 36.6 (físicamente imposible en v/v).

2. **Centinelas grandes de resistividad sin detectar** — Herramientas antiguas codifican nulos como 1e9 o 1e11 Ohm·m. Los centinelas clásicos (-9999, -999.25) no los capturan. RT llegaba a 100,000 Ohm·m en el dataset post-limpieza.

3. **Estrategia de clip destructiva** — La función `clip_physical_ranges()` original usaba la misma acción para features y target: NaN a todo lo fuera de rango, luego `dropna(ALL_COLS)`. Esto eliminaba el 100% de las filas de algunos pozos cuando un solo feature tenía problema de escala (ej. Bieberle_Trust_2: SP en escala absoluta 0–666 mV → todas las filas como NaN → pozo vacío).

4. **Estadísticas del EDA eran incorrectas** — El `documentation/01_eda.md` previo tenía NPHI media = 0.267 v/v, cuando el valor sin corregir era 36.6. EDA corrido sobre datos sin corregir.

## 11:30 — Correcciones implementadas

### `src/data_loader.py`

**Substep 3.2 — Centinelas de resistividad:**
```python
for res_col in ("RT", "RILM"):
    if res_col in out.columns:
        out.loc[out[res_col].abs() > 1e6, res_col] = np.nan
```

**Substep 3.3 — Conversión NPHI % → v/v:**
```python
if "%" in nphi_unit or (len(nphi_nonnull) > 0 and float(nphi_nonnull.median()) > 1.5):
    out["NPHI"] = out["NPHI"] / 100.0
```
Doble guarda: revisa la unidad del LAS y, si es ambigua, usa la mediana como heurístico.

### `src/preprocessing.py`

Separación de `clip_physical_ranges()` en dos funciones con semánticas distintas:

- **`clip_features_to_bounds(df)`** — Clip al borde para features (GR, RT, RILM, NPHI, SP). Preserva filas. Límites amplios para neutralizar spikes sin descartar litologías legítimas.
- **`filter_invalid_target_rows(df)`** — Drop de filas donde DEN está fuera de (1.5, 3.1) g/cc. El target no puede falsificarse.
- **`clip_physical_ranges()`** — Conservado como wrapper backwards-compatible.

Nuevas constantes:
```python
FEATURE_BOUNDS = {"GR": (0, 400), "RT": (0.05, 50000), "RILM": (0.05, 50000),
                  "NPHI": (-0.15, 0.80), "SP": (-1000, 1000)}
TARGET_BOUNDS = (1.5, 3.1)
```

`preprocess_well()` refactorizado con el orden documentado en EDA:
1. clip_features_to_bounds
2. filter_invalid_target_rows
3. apply_log_rt (si log_rt=True)
4. fit_scaler / scaler.transform

## Decisiones de diseño

| Decisión | Elección | Por qué |
|----------|---------|---------|
| Features con outliers | Clip al borde | Preserva la fila; el target DEN sigue siendo válido |
| Target DEN con outliers | Drop de fila | No se puede entrenar con DEN=1.0 (washout) o DEN=4.0 (error) |
| NPHI en % | Dividir por 100 si mediana > 1.5 o unit contiene "%" | Heurístico doble; funciona incluso cuando unit="" |
| Centinelas RT | > 1e6 → NaN | Captura los 1e9/1e11 de herramientas antiguas KGS |
| Normalización | Min-max per-well | Confirmado: cada pozo tiene su propio WellScaler sin contaminación cruzada |

## Errores cometidos

1. **Asumí que NPHI estaba en v/v sin verificar** — Los histogramas del EDA mostraban media ~36, que es imposible en v/v. Debería haber inspeccionado la unidad del LAS al inicio de Phase 0. Perdí un ciclo de EDA corriendo sobre datos en la escala incorrecta.

2. **No consideré pozos con SP en escala absoluta** — El rango [-200, 100] mV para SP era demasiado estricto. Bieberle_Trust_2 tiene SP 0–666 mV (referencia al nivel del mar). La estrategia NaN universal lo dejaba vacío. Aprendizaje: los bounds de features deben ser loose y la normalización per-well absorbe los offsets absolutos.
