Design decisions log

These decisions do not prove anything about the model's results, but they tell how I approached the project: what problem I was facing at each moment, what I thought was causing it, and what I decided to do. This is the foundation that, indirectly, makes the results credible. They are grouped by phase and each card expands on click. If you are looking for the results, they are in the Report.

Phase 0 setup · data · environment

torch outside requirements.txt

Why: the base image pytorch/pytorch:*-cuda* already includes CUDA torch; listing it would install the CPU version on top.

2026-05-28

Canonical mnemonic mapping

Choice: RILD→RT, CNLS/CNPOR→NPHI, RHOB→DEN at load time.

Why: Kraft Prusa uses CNLS for neutron; mapping to canonical names avoids multiplying variants downstream.

2026-05-28

Docker image pytorch 2.4.1-cuda12.1-cudnn9

Why: PyTorch 2.4.x switched to cudnn9; the cudnn8 tag does not exist (found when building).

2026-05-28

mypy.ini with per-package overrides

Choice: central configuration + assert is not None instead of scattered type: ignore.

Why: keeps the code free of tooling annotations and documents invariants.

2026-05-28

Scripts with numeric prefix

Why: the file-naming.md convention requires NN_<name>.py for execution order.

2026-05-28

Phase 1 EDA · preprocessing

log₁₀ on RT and RILM

Why: resistivity is log-normal by physics; log₁₀ linearizes the range (1–10⁴ Ohm·m) and improves the gradient. It is a domain transformation, not just normalization.

2026-05-28

NPHI % → v/v (double heuristic)

Choice: divide by 100 if the unit contains "%" or the median > 1.5.

Why: Kraft Prusa stores CNLS in %; the double guard works even with an empty unit field.

2026-05-29

Resistivity sentinels > 1e6 → NaN

Why: old KGS tools encode nulls as 1e9/1e11 Ohm·m; the classic sentinels (−9999) do not catch them.

2026-05-29

Wide feature bounds + per-well

Why: narrow bounds left wells with SP on an absolute scale (0–666 mV) empty; per-well normalization absorbs those offsets.

2026-05-29

Outliers by consensus voting

Choice: 5 detectors (MAD, IQR, Z, percentile, Isolation Forest), ≥2 → interpolation.

Why: fixed clipping eliminated 100% of rows in wells with systematic anomalies.

2026-05-31

Removal of _part1 duplicates

Choice: delete Weber_'A'_13_part1 and Frees-Burmeister_13_part1.

Why: byte-for-byte identical to their versions without the suffix (duplicate download from KGS) → 30 unique wells, 27 train, 27 folds.

2026-05-31

Per-column normalization (YJ + z-score)

Choice: Yeo-Johnson for GR/SP/DEN (skewed), z-score for RT/RILM/NPHI (symmetric).

Why: choosing the normalizer based on empirical skewness raised the physical $R^2$ from 0.125 to 0.329.

Alternative: uniform min-max (MAE 0.183) and Yeo-Johnson for everything.

2026-06-02

Features clipped, target DEN dropped

Why: a noisy feature is recoverable; a false DEN (washout 1.0, error 4.0) cannot be used as a target.

2026-05-29

Dolecheck_1 documented, not removed

Why: it has NPHI out of range (0.81–5.69 v/v); it is kept clipped and documented as an anomalous case.

2026-05-29

Phase 2 baseline MLP

MLP architecture 5→64→64→32→1

Why: sufficient capacity for ~5k samples per fold without overfitting; Adam + MSE + early stopping.

2026-05-29

Main metric: MAE + PE₉₀

Why: $R^2$ is distorted by outlier wells in a heterogeneous dataset; MAE is more interpretable for the domain.

2026-05-31

λ=0 reproduces the baseline exactly

Why: control condition — the physical term cancels and only the data error remains; guarantees an honest comparison.

2026-05-29

Phase 3 PINN · physics · optimization

Calibration in normalized space

Choice: fit the physical coefficients on post-normalization data, not raw.

Why: $R^2$ rises from 0.125 to 0.329; with Yeo-Johnson+standardize the intercept B=0 by construction (both variables centered at 0).

2026-06-02

Bivariate physics DEN = A·NPHI + D·(NPHI×GR)

Why: the interaction term captures clay content (high GR attenuates the density–neutron slope); $R^2$ 0.330 → 0.338.

Alternative: univariate DEN = A·NPHI.

2026-06-03

DCAL_WEIGHT: physics weighted by caliper

Why: turns off the physics in washout (w→0), making it safe to apply with high λ; produces monotonic improvement up to ≈0.5.

Alternative: uniform physics — degraded wells with washout.

2026-06-03

norm_bounds in WellScaler (anti-NaN)

Why: Yeo-Johnson with negative lambda has a finite domain; clipping predictions to the well's observed range before inverting avoids NaN (affected 13/27 folds).

Alternative: clip to ±5σ hardcoded — less precise than the well's actual range.

2026-06-02

λ = 0.5 operational (robustness over mean)

Why: λ=1.0 lowers the mean MAE but improves only 16 of 27 wells; λ=0.5 improves 22 of 27 capturing almost all the gain.

2026-06-03

Fast config (epochs 300, patience 10)

Why: the bottleneck was patience (post-improvement waiting), not the architecture; cuts the fold time 3× without quality loss.

2026-06-02

GPU-resident + parallel sweep (4.85×)

Why: measured — preprocessing 7%, GPU at 47% starved by per-batch overhead. Data in VRAM (2.18×) + 3 concurrent (GPU 90%) = 4.85×.

Alternative: cache preprocessing (high effort, 7% gain).

2026-06-03

Phase 4 external validation · results

Two external inference protocols

Why: ensemble (27 LOWO) + single model as a robustness analysis; the PINN improves in both → real advantage, not an ensemble artifact.

2026-06-03

external_eval module in src/

Why: the external inference logic is used by 3 scripts (08, 09, 10); by convention it lives in src/ with tests.

2026-06-03

In-sample vs out-of-sample diagnostic

Why: distinguishes capacity (0.88 in-sample) from generalization; the architecture is not the limit, the challenge is generalizing.

2026-06-03

Honest figures (λ=0.5, panel of 3 wells)

Why: a "best improvement" well is usually difficult (both models mediocre) — a bad figure; better a panel of the 3 blind wells with λ consistent with the recommendation.
